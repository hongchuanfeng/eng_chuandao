Base64 Fundamentals

1. What is Base64 encoding

Base64 is a way to represent binary data using 64 printable characters (A-Z, a-z, 0-9, +, /). Since 2^6 = 64, Base64 encodes data in 6-bit units.

2. Why use Base64

In many cases you need to transmit or store binary data as text, for example in URLs, JSON, forms, and emails. Base64 contains only printable characters and avoids the string terminator \0, preventing truncation when binary data is treated as a string.

3. Encoding rules (highlights)

  • Every 3 bytes (24 bits) become 4 groups of 6 bits, each mapping to one Base64 character.
  • If the input length is not a multiple of 3, pad zeros at the end before encoding and use = as padding:
  • Remainder 1 byte ‚Üí end with ==; remainder 2 bytes ‚Üí end with =.

4. Examples

Example 1: bytes 0 1 2
Binary: 0011 0000 ¬∑ 0011 0001 ¬∑ 0011 0010 ‚Üí group by 6 bits ‚Üí 12, 3, 4, 50 ‚Üí chars M D E y
Result: MDEy

Example 2 (with padding): bytes 0 1 2 3
Groups: 12, 3, 4, 50, 12, 48 (zero padded at the end) ‚Üí chars M D E y M w
Result: MDEyMw==
